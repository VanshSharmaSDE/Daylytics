## ⚠️ Remaining Limitations

### 📋 **File System Constraints**

> **Note:** Version 2.0 solved the major performance issues (slow loading, stutter, lag) but some feature limitations remain:

#### **1. No File Recovery System**
- **Status:** Deleted files are permanently removed
- **Workaround:** Be careful when deleting; confirmation modals help prevent accidents

#### **2. Folder Deletion Restrictions**
- **Status:** Folders must be empty to delete
- **Workaround:** Delete all nested content first, then remove folder

#### **3. No Bulk Operations**
- **Status:** Cannot multi-select files/folders
- **Workaround:** Delete or move items one at a time

#### **4. Limited Categorization**
- **Status:** Files exist in one folder only (no tags or labels)
- **Workaround:** Use descriptive naming conventions

#### **5. No File Version History**
- **Status:** Edits overwrite previous content
- **Workaround:** Manually duplicate files before major edits
#### **6. No Content Search**
- **Status:** Search matches titles/names only, not file content
- **Workaround:** Use descriptive, keyword-rich file titles

#### **7. No File Export**
- **Status:** Cannot download files as .md, .txt, or .pdf
- **Workaround:** Copy/paste content manually

> **✅ SOLVED in v2.0:** Slow folder loading, UI stutter, lag, and performance issues that existed in v1.x

## 🏃 Running Locally

### Prerequisites
- **Node.js** (v14 or higher)
- **MongoDB** (local installation or MongoDB Atlas cloud)
- **npm** or **yarn** package manager

### Setup Instructions

1. **Clone the Repository**
   ```bash
   git clone <your-repo-url>
   cd Daylytics
   ```

2. **Backend Setup**
   ```powershell
   cd server
   npm install
   ```
   
   Create `.env` file in `server/` directory:
   ```env
   MONGO_URI=mongodb://localhost:27017/daylytics
   JWT_SECRET=your_super_secret_key_change_in_production
   PORT=5000
   BACKEND_URL=http://localhost:5000/api/health
   ```

3. **Frontend Setup**
   ```powershell
   cd client
   npm install
   ```
   
   Create `.env` file in `client/` directory (optional):
   ```env
   VITE_API_URL=http://localhost:5000
   ```

4. **Start Development Servers**
   
   **Terminal 1 - Backend Server:**
   ```powershell
   cd server
   npm run dev
   ```
   Backend runs on `http://localhost:5000`
   
   **Terminal 2 - Frontend Client:**
   ```powershell
   cd client
   npm run dev
   ```
   Frontend runs on `http://localhost:5173`

5. **Access the Application**
   - Open your browser to `http://localhost:5173`
   - Register a new account or use seeded test credentials

### Optional: Load Test Data
```powershell
cd server
npm run seed
```
Creates test account:
- **Email:** `test@example.com`
- **Password:** `password`
- **Sample Data:** 4 tasks from yesterday (2 completed, 2 pending)

## 📚 API Documentation

### Authentication Endpoints
- `POST /api/auth/register` - Register new user account
- `POST /api/auth/login` - Login and receive JWT token
- `GET /api/auth/me` - Get current authenticated user
- `PUT /api/auth/profile` - Update user name/email
- `PUT /api/auth/password` - Change password

### Task Endpoints
- `GET /api/tasks?date=YYYY-MM-DD` - Get tasks for specific date
- `POST /api/tasks` - Create new task with optional auto-add setting
- `PUT /api/tasks/:id` - Edit task title
- `PATCH /api/tasks/:id` - Toggle task completion status
- `PATCH /api/tasks/:id/auto-add` - Toggle auto-add to next day setting (**New in v2.0**)
- `DELETE /api/tasks/:id` - Delete single task
- `DELETE /api/tasks?date=YYYY-MM-DD` - Delete all tasks for date

### Archive Endpoints
- `POST /api/archive/rollover?date=YYYY-MM-DD` - Manually archive date
- `GET /api/archive` - Get all archived days with statistics
- **Note:** Auto-archive runs at midnight daily (no manual trigger needed)

### File Endpoints
- `GET /api/files` - Get all files for current user (optionally filter by folder)
- `GET /api/files/:id` - Get specific file by ID
- `POST /api/files` - Create new file (with optional folder assignment)
- `PUT /api/files/:id` - Update file title/content
- `DELETE /api/files/:id` - Delete file permanently
- `PATCH /api/files/:id/pin` - Toggle file pin status

### Folder Endpoints
- `GET /api/folders` - Get all folders (optionally filter by parentFolder)
- `GET /api/folders/:id` - Get specific folder details
- `POST /api/folders` - Create new folder (with optional parent)
- `PUT /api/folders/:id` - Update folder name/location
- `DELETE /api/folders/:id` - Delete folder (must be empty)
- `PATCH /api/folders/:id/pin` - Toggle folder pin status

### FlowChart Endpoints (**New in v2.0**)
- `GET /api/flowcharts` - Get all flowcharts for current user
- `GET /api/flowcharts/:id` - Get specific flowchart by ID
- `POST /api/flowcharts` - Create new flowchart with title, nodes, edges, viewport
- `PUT /api/flowcharts/:id` - Update flowchart data
- `DELETE /api/flowcharts/:id` - Delete flowchart permanently

### FlowChart Share Endpoints (**New in v2.0**)
- `GET /api/flowchart-shares` - Get all flowchart shares for current user
- `POST /api/flowchart-shares` - Create new share with expiration and limits
- `GET /api/flowchart-shares/:token` - Get shared flowcharts (public, no auth)
- `DELETE /api/flowchart-shares/:id` - Delete share
- `PATCH /api/flowchart-shares/:id/toggle` - Toggle share active status

### Planner Endpoints (**New in v2.0**)
- `GET /api/planner/:weekStart` - Get plans for specific week (YYYY-MM-DD format)
- `PUT /api/planner/:weekStart` - Save/update plans for specific week (7 days)
